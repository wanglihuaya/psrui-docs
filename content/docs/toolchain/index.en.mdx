## How to read this section

- first ask what the tool does upstream
- then check the `PSRUI today` note
- then decide whether the workflow belongs in the GUI or back in the CLI

## Scope

- focused on archive processing, TOA extraction, calibration, timing, and the surrounding front-end/back-end chain
- intentionally not a full reproduction of backend-specific flags, obscure options, or every plugin in the upstream manuals

If you want one page that links the original PSRCHIVE paper to the rest of this site, start with [Pulsar Data Analysis with PSRCHIVE](./pulsar-data-analysis-with-psrchive).
