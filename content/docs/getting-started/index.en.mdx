## What this section answers

- who PSRUI is for, and what it does not try to replace
- when to choose the local runtime versus the Docker runtime
- what the shortest path is from opening an archive to exporting a processed result

> **PSRUI today**: archive previews, channel zapping, live `pam` controls, `pat` TOA extraction, `pac` calibration previews, and batch recipes.

> **Common misunderstanding**: PSRUI is not yet a full `tempo2` residual-analysis GUI, and it is not a `dspsr` frontend for converting raw baseband data into folded archives.
